About EMMSAP

The Electronic Medieval Music Score Archive Project (EMMSAP) is a project developed by Michael Scott Cuthbert with Anna Grau and others to encode all polyphonic European music composed between 1300 and 1430 and, using tools from music21 and STAMR, to make this repertory searchable and able to be manipulated. This search engine uses tools from an early "Atlas" prototype (announced at the Schola Cantorum Basilensis) together with the STAMR framework.

About STAMR

"Small Tools for Agile Music Research" is a project run at MIT by Michael Scott Cuthbert with support from the Seaver Institute. It attempts to make the development of music research tools faster by producing a reusable framework on which to build research tools that can be adapted quickly to support changing research needs. Uses j2p2j for most purposes.

cuthbertLab and the Seaver Institute

cuthbertLab is a research lab on computational musicology, medieval music, and the digital humanities run by Michael Scott Cuthbert at MIT. It is most well known for the creation of the music21 toolkit for computational music research.

The Seaver Institute, located in Los Angeles, provides seed money to highly regarded organizations, including MIT, for particular projects which offer the potential for significant advancement in their fields. They have supported the creation of STAMR and music21.

Tiny Notation Reference

  1. Note names are: a,b,c,d,e,f,g and r for rest
  2. Flats, sharps, and naturals are notated as #,- (not b), and (if needed) n.
  3. Note octaves are specified as follows::
    • CC to BB = from C below bass clef to second-line B in bass clef
    • C to B = from bass clef C to B below middle C.
    • c to b = from middle C to the middle of treble clef
    • c' to b' = from C in treble clef to B above treble clef
  4. After the note name, a number may be placed indicating the note length: 1 = whole note, 2 = half, 4 = quarter, 8 = eighth, 16 = sixteenth. etc. If the number is omitted then it is assumed to be the same as the previous note. I.e., c8 B c d is a string of eighth notes.
  5. After the number, a ~ can be placed to show a tie to the next note. A "." indicates a dotted note.
  6. For triplets use this notation: "trip{c4 d8}" indicating that these two notes both have "3s" over them. For 4 in the place of 3, use "quad{c16 d e8}". No other tuplets are supported.
